2. RINGKASAN AWAM 1 HALAMAN

FLORA-AI adalah aplikasi web yang membantu pengguna memilih tanaman hias yang cocok dengan kondisi rumah atau ruangannya. Pengguna tidak perlu memahami tanaman satu per satu. Pengguna cukup menjawab beberapa pertanyaan singkat, misalnya kondisi cahaya, seberapa sering bisa menyiram, apakah tanaman diletakkan indoor atau outdoor, besar ruang yang tersedia, budget, dan apakah ada hewan peliharaan.

Setelah jawaban dikirim, FLORA-AI mencocokkan jawaban tersebut dengan dataset tanaman. Dataset itu berupa file Excel bernama References/dataset_FLORA.xlsx. Di dalamnya ada daftar tanaman, daftar pertanyaan, dan daftar aturan rekomendasi. Aturan ini mirip cara berpikir "jika kondisi A dan B terpenuhi, maka tanaman X cocok". Misalnya, jika cahaya rendah, penyiraman jarang, posisi indoor, ruang kecil, perawatan mudah, budget rendah, jenis tampilan daun, dan ukuran kecil, maka sistem dapat merekomendasikan Sansevieria.

Bagian "AI" pada FLORA-AI bukan berarti program ini melatih model machine learning seperti neural network. Dari pemeriksaan kode, tidak ditemukan proses training model, tidak ada epoch, tidak ada split data training-testing, dan tidak ada perhitungan akurasi model. AI di sini lebih tepat dijelaskan sebagai sistem pakar atau sistem rekomendasi berbasis aturan. Sistem seperti ini tetap termasuk pendekatan kecerdasan buatan klasik karena komputer memakai aturan untuk meniru proses pengambilan keputusan sederhana.

Program web dibuat dengan Flask dan Python. Flask menangani halaman web, route, form, dan tampilan HTML. Python di file ai/recommender.py menangani logika rekomendasi. Dataset dibaca memakai pandas dan openpyxl. Saat pengguna membuka halaman rekomendasi, daftar pertanyaan diambil dari dataset. Saat pengguna mengirim jawaban, program memvalidasi jawaban, mencocokkan aturan, mengurutkan hasil paling cocok, lalu menampilkan maksimal 3 tanaman rekomendasi.

Kelebihan pendekatan ini adalah hasilnya mudah dijelaskan saat presentasi. Kita bisa menunjukkan alasan kenapa tanaman direkomendasikan, karena keputusan berasal dari rules yang bisa dibaca. Kekurangannya, sistem tidak otomatis belajar sendiri. Jika ingin menambah tanaman atau memperbaiki hasil, dataset dan rules perlu diperbarui secara manual. Jadi kualitas rekomendasi sangat bergantung pada kelengkapan dan konsistensi dataset.


3. APA SEBENARNYA "AI" DI FLORA-AI?

FLORA-AI lebih tepat disebut sistem rekomendasi berbasis rule/forward logic. Maksudnya, program mengambil fakta dari jawaban pengguna, lalu mencocokkannya dengan aturan di dataset.

Contoh sederhana:
- Fakta dari pengguna: cahaya rendah, penyiraman jarang, indoor, ruang kecil.
- Rule di dataset: jika beberapa kondisi tersebut terpenuhi, hasilnya Sansevieria.
- Output: sistem menampilkan Sansevieria sebagai rekomendasi.

Bedanya dengan machine learning:
- Machine learning biasanya melatih model dari data latih, lalu model belajar pola secara statistik.
- FLORA-AI tidak melatih model. FLORA-AI membaca dataset Excel dan mencocokkan rules.
- Tidak ditemukan file model, training script, epoch, loss, akurasi model, train-test split, atau library ML seperti scikit-learn, TensorFlow, atau PyTorch.

Bukti:
- requirements.txt hanya memuat Flask, pandas, openpyxl, gunicorn, dan waitress untuk aplikasi utama.
- ai/recommender.py memakai pandas.ExcelFile, pd.read_excel, validasi dataset, parsing rule, dan ranking rekomendasi.
- tests/test_recommender.py menguji rule-based behavior seperti wildcard, parsing kondisi, validasi dataset, dan hasil rekomendasi.

Kesimpulan aman untuk presentasi:
"AI pada FLORA-AI adalah AI berbasis sistem pakar sederhana. Program ini menggunakan forward logic/rule-based reasoning, bukan machine learning yang dilatih."

5. ALUR PROGRAM DARI PENGGUNA

1. Pengguna membuka https://flora-ai-ochre.vercel.app/.
2. Halaman Beranda menampilkan identitas FLORA dan tombol "Mulai Rekomendasi".
3. Pengguna masuk ke halaman /rekomendasi.
4. Pengguna melihat form preferensi tanaman dengan progress 1 / 10.
5. Pengguna menjawab 10 pertanyaan:
   - kondisi cahaya,
   - frekuensi penyiraman,
   - posisi tanaman,
   - luas ruang,
   - tingkat perawatan,
   - ada/tidak hewan peliharaan,
   - nyaman/tidak dengan duri,
   - budget,
   - jenis tampilan,
   - ukuran tanaman.

7. Server memeriksa apakah semua jawaban valid.
8. Jika ada jawaban kosong atau tidak valid, halaman menampilkan pesan error.
9. Jika jawaban valid, server mencocokkan jawaban dengan rules di dataset.
10. Sistem memilih tanaman paling cocok, lalu membuat alasan rekomendasi.
11. Pengguna melihat hasil berupa tanaman rekomendasi, foto, alasan cocok, dan catatan singkat.

7. PENJELASAN KODE PYTHON PER BAGIAN

A. app.py

1. Import library
app.py mengimpor os, Flask, redirect, render_template, request, url_for, serta fungsi dari ai.recommender. Ini menunjukkan app.py tidak menyimpan logika rekomendasi utama, tetapi memanggil modul rekomendasi terpisah.

2. app = Flask(__name__)
Baris ini membuat objek aplikasi Flask. Objek ini menjadi pusat route dan konfigurasi aplikasi.

3. REFERENCES
REFERENCES adalah daftar referensi yang ditampilkan di halaman Tentang. Isinya tiga referensi: dua jurnal forward chaining dan satu referensi tanaman indoor.

4. MEMBERS
MEMBERS berisi data anggota kelompok: nama, NIM, kelas, dan peran.

5. SECURITY_HEADERS
Bagian ini berisi header keamanan seperti Content-Security-Policy, X-Content-Type-Options, Referrer-Policy, dan Permissions-Policy. Fungsinya membatasi sumber script/style/gambar dan mengurangi risiko tertentu. Ini bukan berarti aplikasi sempurna aman, tetapi ada upaya dasar keamanan HTTP.

6. inject_globals()
Fungsi ini memakai @app.context_processor. Artinya data yang dikembalikan fungsi ini bisa tersedia di template. Data yang dimasukkan adalah app_name, tagline, dan dataset_stats. dataset_stats diambil dari get_dataset_stats().

7. add_security_headers(response)
Fungsi ini memakai @app.after_request. Artinya setelah route menghasilkan response, Flask menjalankan fungsi ini untuk menambahkan security headers.

8. route "/"
Fungsi beranda() merender templates/beranda.html. Halaman ini menjadi halaman awal aplikasi.

9. route "/favicon.ico"
Fungsi favicon() mengalihkan request favicon ke static/img/icon.png.

10. route "/healthz"
Fungsi healthz() mengambil statistik dataset, lalu mengembalikan JSON sederhana: status ok, jumlah plants, dan jumlah questions. Test lokal memastikan hasilnya {"status": "ok", "plants": 30, "questions": 10}.

11. route "/rekomendasi" GET dan POST
Fungsi rekomendasi() adalah alur utama:
- Saat GET, program memanggil get_questions() dan menampilkan form.
- Saat POST, program membaca request.form berdasarkan variabel pertanyaan.
- Jawaban divalidasi dengan validate_answers().
- Jika tidak ada error, program memanggil recommend_plants().
- Hasil, error, questions, dan answers dikirim ke templates/rekomendasi.html.

12. route "/tentang"
Fungsi tentang() merender templates/tentang.html dengan references dan members.

13. if __name__ == "__main__"
Bagian ini menjalankan app secara lokal jika file app.py dieksekusi langsung. Debug aktif jika environment variable FLASK_DEBUG bernilai "1".


B. ai/recommender.py

1. DATASET_PATH dan STATIC_DIR
DATASET_PATH menunjuk ke References/dataset_FLORA.xlsx. STATIC_DIR menunjuk ke folder static. Dua path ini dipakai untuk membaca dataset dan mencari gambar tanaman.

2. REQUIRED_SHEETS
REQUIRED_SHEETS berisi tiga sheet wajib: tanaman, pertanyaan, rules. Jika salah satu sheet tidak ada, _load_dataset() akan menaikkan ValueError.

3. REQUIRED_PLANT_COLUMNS, REQUIRED_QUESTION_COLUMNS, REQUIRED_RULE_COLUMNS
Bagian ini mendefinisikan kolom wajib di setiap sheet. Tujuannya menjaga dataset tidak rusak atau kurang kolom.

4. FREE_VALUES
FREE_VALUES berisi "bebas" dan "tidak_masalah". Dalam _condition_matches(), nilai ini diperlakukan sebagai wildcard. Artinya jika actual atau expected adalah nilai bebas, kondisi dianggap cocok.

5. Variabel hewan dan duri
PET_VARIABLE = "hewan_peliharaan".
THORN_COMFORT_VARIABLE = "nyaman_duri".
THORN_PLANT_ATTRIBUTE = "berduri_tajam".
Tiga konstanta ini membuat logika khusus untuk rumah dengan hewan dan kenyamanan terhadap duri.

6. MAX_RECOMMENDATIONS dan MAX_REASON_CHIPS
MAX_RECOMMENDATIONS = 3 membatasi hasil maksimal tiga tanaman. MAX_REASON_CHIPS = 4 membatasi alasan singkat maksimal empat item.

7. _normalize()
Fungsi ini membersihkan nilai: mengubah NaN menjadi string kosong, menghapus spasi depan/belakang, dan mengubah teks menjadi huruf kecil. Ini penting agar "Rendah" dan "rendah" diperlakukan sama.

8. _title_value()
Fungsi ini membuat label tampilan dari nilai dataset, misalnya "rendah" menjadi "Rendah" dan "tidak_masalah" menjadi "Tidak Masalah".

9. _split_choices()
Fungsi ini memecah pilihan seperti "rendah|sedang|tinggi" menjadi list ["rendah", "sedang", "tinggi"].

10. _load_dataset()
Fungsi ini membaca dataset Excel. Langkahnya:
- membuka file Excel dengan pandas.ExcelFile,
- mengecek sheet wajib,
- membaca sheet tanaman, pertanyaan, rules dengan pd.read_excel,
- mengisi nilai kosong dengan fillna(""),
- memvalidasi kolom,
- membuat peta kode/nama tanaman,
- memvalidasi rules.

Fungsi ini memakai @lru_cache(maxsize=1), sehingga hasil pembacaan dataset disimpan. Jika fungsi dipanggil berulang, program tidak membaca Excel dari awal terus-menerus.

11. _validate_columns()
Fungsi ini memastikan setiap sheet memiliki kolom wajib. Jika ada kolom hilang, program memberi error yang jelas.

12. _build_plant_key_map()
Fungsi ini membuat peta dari kode tanaman, nama tanaman, dan nama umum lain ke kode tanaman. Contoh: Sansevieria dan Snake Plant bisa diarahkan ke kode T001.

13. _build_rule_display_names()
Fungsi ini membuat peta kode tanaman ke nama tanaman yang akan ditampilkan.

14. _validate_dataset()
Fungsi ini memastikan rules valid:
- variabel di rules harus ada di sheet pertanyaan,
- nilai di rules harus ada di pilihan pertanyaan,
- hasil rules harus merujuk tanaman yang ada di sheet tanaman.

15. get_dataset_stats()
Fungsi ini mengembalikan jumlah tanaman, jumlah pertanyaan, dan jumlah rules. Dari dataset saat ini: 30 tanaman, 10 pertanyaan, 63 rules.

16. get_questions()
Fungsi ini mengubah sheet pertanyaan menjadi format list dictionary yang mudah dipakai template. Setiap pertanyaan memiliki code, variable, question, choices, dan step.

17. validate_answers()
Fungsi ini memastikan setiap pertanyaan sudah dijawab dan nilai jawaban termasuk pilihan yang diizinkan. Jika kosong, muncul error "belum dipilih". Jika nilai tidak ada di pilihan, muncul error "tidak valid".

18. recommend_plants()
Fungsi ini menjalankan proses rekomendasi utama:
- normalisasi jawaban,
- evaluasi semua rules,
- pilih rules yang aktif atau semua rules jika tidak ada rule aktif,
- urutkan berdasarkan matched_count, attribute_matches, total_conditions,
- bangun daftar rekomendasi,
- bangun summary.

19. _evaluate_rules()
Fungsi ini membaca setiap rule, memecah kondisi, mencari tanaman hasil, menghitung kondisi yang cocok, menghitung kecocokan atribut tanaman, lalu menandai apakah rule aktif penuh.

20. _condition_matches()
Fungsi ini membandingkan jawaban dengan nilai rule. Jika salah satunya "bebas" atau "tidak_masalah", hasilnya cocok.

21. _parse_conditions()
Fungsi ini memecah teks kondisi dengan " AND ". Contoh:
"cahaya=rendah AND jenis_tampilan=daun" menjadi dictionary {"cahaya": "rendah", "jenis_tampilan": "daun"}.

22. _resolve_plant_key()
Fungsi ini mencari kode tanaman dari hasil rule. Hasil rule bisa berupa kode, nama tanaman, atau nama umum lain.

23. _get_plant_by_key()
Fungsi ini mengambil baris tanaman berdasarkan kode tanaman.

24. _build_recommendations()
Fungsi ini membuat daftar hasil akhir. Setiap item punya rank, status, name, description, long_description, care, reasons, badges, image_filename, dan matched_rule. Fungsi ini juga menghindari tanaman duplikat dan berhenti setelah mencapai MAX_RECOMMENDATIONS.

25. _count_plant_attributes()
Fungsi ini menghitung atribut tanaman yang sesuai dengan jawaban pengguna. Untuk hewan dan duri, ada perlakuan khusus.

26. _build_reason_list() dan _reason_for_field()
Fungsi ini membangun alasan singkat kenapa tanaman cocok. Contohnya: "Tahan di tempat minim cahaya", "Ideal untuk dalam ruangan", "Tidak perlu sering disiram".

27. _build_badges()
Fungsi ini membuat label atribut tanaman seperti "Cahaya: Rendah" atau "Budget: Rendah". Namun test lokal memastikan tampilan akhir tidak menampilkan bagian "Atribut tanaman" di halaman hasil.

28. _build_long_description()
Fungsi ini membuat deskripsi panjang yang lebih natural untuk hasil rekomendasi.

29. _image_for_plant()
Fungsi ini membuat slug nama tanaman, lalu mencari file gambar di static/img/flowers dengan ekstensi .jpg, .jpeg, .png, atau .webp. Jika tidak ditemukan, sistem memakai img/flowers/default.jpg.

30. _build_summary()
Fungsi ini membuat ringkasan hasil, misalnya "Sansevieria menjadi pilihan utama untuk kondisi cahaya Rendah dan posisi Indoor."


8. PENJELASAN DATASET

Path dataset:
References/dataset_FLORA.xlsx

Daftar sheet:
1. tanaman
2. pertanyaan
3. rules

Jumlah baris:
- tanaman: 30 baris
- pertanyaan: 10 baris

A. Sheet tanaman

Sheet tanaman adalah "database tanaman". Setiap baris mewakili satu tanaman hias.

Kolom pada sheet tanaman:
- kode_tanaman: kode unik tanaman, misalnya T001.
- nama_tanaman: nama utama tanaman, misalnya Sansevieria.
- nama_umum_lain: nama lain, misalnya Snake Plant.
- deskripsi_singkat: penjelasan singkat tanaman.
- cahaya: kebutuhan cahaya, misalnya rendah/sedang/tinggi.
- penyiraman: kebutuhan penyiraman, misalnya jarang/sedang/sering.
- posisi: indoor atau outdoor.
- ruang: kecil/sedang/besar.
- perawatan: mudah/sedang/sulit.
- pet_safe: apakah relatif aman untuk rumah dengan hewan, ya/tidak.
- berduri_tajam: apakah tanaman berduri atau tajam, ya/tidak.
- budget: rendah/sedang/tinggi.
- jenis_tampilan: daun/bunga/kaktus.
- ukuran: kecil/sedang/besar.
- alasan_rekomendasi: catatan kenapa tanaman cocok.
- tips_perawatan: tips perawatan di dataset, tetapi pada template hasil final yang dibaca tidak terlihat dipakai langsung.

Contoh 3 baris awal sheet tanaman:
1. T001 | Sansevieria | Snake Plant | cahaya rendah | penyiraman jarang | posisi indoor | ruang kecil | perawatan mudah | budget rendah | jenis daun | ukuran kecil.
2. T002 | Zamia Dollar | ZZ Plant | cahaya rendah | penyiraman jarang | posisi indoor | ruang kecil | perawatan mudah | budget sedang | jenis daun | ukuran kecil.
3. T003 | Sirih Gading | Pothos | cahaya rendah | penyiraman sedang | posisi indoor | ruang kecil | perawatan mudah | budget rendah | jenis daun | ukuran kecil.

Daftar tanaman:
Sansevieria, Zamia Dollar, Sirih Gading, Lili Paris, Lili Perdamaian, Monstera, Karet Kebo, Pakis Boston, Lidah Buaya, Palem Bambu, Kaktus Mini, Aglaonema, Melati, Mawar, Lavender, Anggrek Bulan, Bougenville, Anthurium, Puring, Calathea, Philodendron, Palem Kuning, Paku Tanduk Rusa, Begonia, Tapak Dara, Keladi Hias, Peperomia, Fittonia, Kembang Sepatu, Coleus.

B. Sheet pertanyaan

Sheet pertanyaan adalah "daftar pertanyaan yang ditampilkan di form".

Kolom pada sheet pertanyaan:
- kode_pertanyaan: kode pertanyaan, misalnya P001.
- variabel: nama variabel yang dipakai kode dan rules.
- pertanyaan: teks pertanyaan untuk pengguna.
- pilihan: pilihan jawaban, dipisahkan dengan tanda "|".
- keterangan: penjelasan arti pilihan.

Daftar variabel pertanyaan dan pilihan:
1. cahaya: rendah|sedang|tinggi
2. penyiraman: jarang|sedang|sering
3. posisi: indoor|outdoor
4. ruang: kecil|sedang|besar
5. perawatan: mudah|sedang|sulit
6. hewan_peliharaan: ya|tidak
7. nyaman_duri: ya|tidak
8. budget: rendah|sedang|tinggi
9. jenis_tampilan: bebas|daun|bunga|kaktus
10. ukuran: kecil|sedang|besar

Contoh 3 baris awal sheet pertanyaan:
1. P001 | cahaya | Bagaimana kondisi cahaya di tempat tanaman akan diletakkan? | rendah|sedang|tinggi.
2. P002 | penyiraman | Seberapa sering kamu bisa menyiram tanaman? | jarang|sedang|sering.
3. P003 | posisi | Di mana tanaman akan diletakkan? | indoor|outdoor.

C. Sheet rules

Sheet rules adalah "aturan IF-THEN". Kolomnya:
- kode_rule: kode unik rule, misalnya R001.
- kondisi: daftar kondisi yang harus dicocokkan, dipisahkan dengan " AND ".
- hasil: nama tanaman yang direkomendasikan jika rule cocok.

Contoh 3 baris awal sheet rules:
1. R001 -> kondisi lengkap dengan cahaya=rendah, penyiraman=jarang, posisi=indoor, ruang=kecil, perawatan=mudah, budget=rendah, ukuran=kecil, jenis_tampilan=daun, hewan_peliharaan=tidak, nyaman_duri=tidak -> hasil Sansevieria.
2. R002 -> kondisi mirip, tetapi budget=sedang -> hasil Zamia Dollar.
3. R003 -> penyiraman=sedang -> hasil Sirih Gading.

D. Hubungan antar sheet

Sheet pertanyaan menentukan variabel dan pilihan jawaban yang boleh dipilih pengguna. Sheet rules memakai variabel yang sama pada kolom kondisi. Sheet tanaman menyimpan atribut tanaman dan data tampilan. Jadi alurnya:

pertanyaan -> jawaban pengguna -> rules -> hasil tanaman -> detail tanaman dari sheet tanaman -> tampilan hasil

E. Efek penggunaan dataset

1. Jika dataset lengkap dan konsisten, hasil rekomendasi lebih mudah dipahami.
2. Jika ada variabel rule yang tidak ada di pertanyaan, _validate_dataset() akan menolak dataset.
3. Jika hasil rule tidak ada di sheet tanaman, _validate_dataset() akan menolak dataset.
4. Jika pilihan pertanyaan terlalu sederhana, hasil rekomendasi juga terbatas.
5. Jika rules kurang lengkap, sistem tetap bisa memilih alternatif terbaik berdasarkan jumlah kondisi yang cocok.

7. Karena tidak belajar otomatis, perubahan pengetahuan harus dilakukan dengan mengubah dataset/rules.

10. EFEK DAN KONSEKUENSI MEMAKAI DATASET/RULE-BASED SYSTEM

A. Kelebihan

1. Mudah dijelaskan
Setiap rekomendasi bisa ditelusuri ke rule dan atribut dataset.

2. Cocok untuk presentasi
Dosen bisa melihat alur dari jawaban, rules, sampai hasil.

3. Tidak butuh dataset besar untuk menjalankan rules
Karena tidak melatih model ML, sistem bisa berjalan dengan dataset kecil selama rules dibuat rapi.

4. Kontrol penuh
Jika ingin tanaman tertentu muncul untuk kondisi tertentu, rules bisa diubah.

5. Hasil konsisten
Input yang sama akan menghasilkan proses rekomendasi yang sama.

B. Kekurangan

1. Tidak belajar otomatis
Sistem tidak memperbaiki dirinya sendiri dari feedback pengguna.

2. Bergantung pada kualitas dataset
Jika atribut tanaman salah, hasil rekomendasi bisa salah.

3. Rules bisa tidak lengkap
Jika kombinasi jawaban tidak ada rule aktifnya, sistem memilih alternatif terbaik, bukan kecocokan penuh.

4. Butuh perawatan manual
Menambah tanaman atau memperbaiki rules harus dilakukan melalui dataset.

5. Pilihan terbatas
Karena pertanyaan hanya 10 dan pilihan sudah ditentukan, preferensi pengguna yang lebih kompleks belum tertangkap.

C. Dampak jika dataset diperbarui

Jika menambah tanaman baru:
- Tambahkan baris di sheet tanaman.
- Tambahkan rules yang menghasilkan tanaman tersebut.
- Pastikan gambar tersedia di static/img/flowers atau fallback akan dipakai.

Jika mengubah pilihan pertanyaan:
- Ubah sheet pertanyaan.
- Ubah rules agar nilai kondisi tetap valid.
- Jalankan test agar _validate_dataset() tidak error.

Jika mengubah rules:
- Hasil rekomendasi bisa berubah.
- Test yang mengharapkan Sansevieria untuk jawaban tertentu mungkin perlu diperbarui jika perubahan memang disengaja.
